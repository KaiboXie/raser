## 调研报告的目标 

- 明确科学研究所要达成的目标
	- 解决怎样的科学问题
	- 什么是通向解决此问题的”拦路虎“
	- 可能伴随的发现会是什么？ 

- 了解达成目标所需要的科学方法
	- 使用怎样的科学装置 
		- 大科学、对撞机、粒子探测器
	- 数据采集与分析的方法
		- 海量数据、人工智能算法

> 最终落实到==我==能在此过程中贡献什么？ 从而拓展人类的认知 

---
##  调研报告的基石 - 文献阅读 

- 文献阅读是推动科研产出的必经之路 
- 有效的阅读是科技文章的关键
	- 科技文章 $\neq$ 教科书 
	- 泛读 + 精读 相结合
	- 带着自己的==问题==进行阅读
- 构建自己专属的”图书馆“ 
	- 收集文献、进行批注
	- 方便未来随时查找 
	- 推荐软件：[[Zotero]] （下载，群讨论） 
	
>  文献阅读的目标： 形成==自己==对于某一个问题的全面而深入的理解
	
---
## 调研报告的主体 - 工作进展报告

- 科研工作的特点是”长跑“
	- 长期持续发力
	- 某些问题需要反复推敲
- 坚持固定周期的进展报告 
	- 最好每周都有小的进展 （即使产生出新的错误） 
	- 超过两周以上，基本上很难有产出
	- ”不怕慢就怕站“ -> 丢掉“憋大招”的幻想！  

---
## 工作进展报告的核心 

- 要解决的问题是什么？ 
	- 与小团队、大团队的关联 
- 采取的方法是什么？ 
- 是否得到了预期的结果
	- 如果得到了，怎样证明是正确的？ （可重复） 
	- 如果没有， 遇到了怎样的新问题，有何想法？ 

>  每周进展报告的目的是==暴露问题==， 借助群体智慧，推进工作 

---
## 总结与讨论的演练

- 明天上午的练习
- 每人准备三页以内的报告
	- 学到的新知识、激发的新想法
	- 遇到的问题，有什么要求和建议
	- 未来的计划 
- 推送到  [[A7 总结与讨论]]

## 更新与合并操作

- 在你的本地电脑安装 [[git]] 、[[VSCode]] 和 [[Obsidian]] 
- 在VSCode 的命令行里进入到工作目录： (例如 D:\\Work )
	- > git clone https://code.ihep.ac.cn/user/raser.git 
	- > cd raser 
	- > git remote add raser https://code.ihep.ac.cn/raser/raser.git 
	- > git config user.name "User Name"
	- > git config user.email "user@email"
	- > git remote update 
	- > git merge raser/main 
- 进行更新文件，保存推送 
	- > git add CHANGED_FILE 
	- > git commit -m" 修改的信息" 
	- > git push 
- 打开浏览器，提交Merge Request
