## Git仓库的作用
组内公开的资源平台，所有人共同使用共同维护
### Git仓库里的内容
- note文件夹
	- 个人进展记录
	- 会议记录
	- 项目进展记录
	- 硬件使用记录
- 其它文件夹
	- 各种程序

### Git仓库的使用
- note文件夹下的md文件
	使用Obisdian打开note文件夹（具体使用方法后续介绍）
- 获取最新的内容
	- 打开任意命令行窗口（建议使用VScode里的终端，以下操作均以VScode为例）
	- 进入到git仓库所在路径
		- `pwd` 查看当前所在路径
		- `ls` 查看当前路径下的文件
		- `cd 相对路径/绝对路径` 跳转路径
	- 在终端中依次输入
		`git status` 查看修改过的文件
		`git stash` 将自己修改过的文件暂时保存至虚拟空间
		`git remote update` 将主仓库更新的文件放入缓存
		`git merge raser/main` 用更新至缓存的文件覆盖本地仓库里的文件
		`git stash pop` 将暂存的至虚拟空间释放
	- 如果释放后修改过的文件与更新后的文件有冲突会有提示，需要自己手动修改

### Git仓库的维护
-  仓库中文件的管理方式
	- 在本地的Git仓库中修改文件
	- 将本地Git仓库中的修改推送至自己的Git仓库
	- 在RASER主仓库中提交合并申请，将自己Git仓库的修改合并至主仓库
- 本地Git仓库的文件管理
	- 使用VScode打开Git仓库所在的文件夹
	- VScode左侧的工具栏中源代码管理将会显示Git仓库中所有文件的改动![capture_20230515155640171.bmp](https://raser-1314796952.cos.ap-beijing.myqcloud.com/capture_20230515155640171.bmp)
- 将本地的修改推送至自己的Git仓库
	- 确保自己是在最新的主仓库的分支的基础上进行的修改（推送前按照之前获取最新内容的方法更新一下本地仓库）
	- 点击上图更改中点击自己想要推送的文件右侧的加号，文件将从更改下移至暂存的更改下
	- 在提交按钮上的文本框输入内容（必须有），点击提交
	- 打开终端，进入到git仓库所在路径，输入`git push`
	- 更新已推送至自己的Git仓库
- 将自己仓库的更新合并至主仓库
	- 进入高能所GitLab
	- 进入主仓库RASER/raser
	- 点击左侧Merge requests
	- 找到New merge request
	- 提交后等待审批


# Obisdian

将本地Git仓库中的note文件夹在Obisdian中打开

## 插件的安装与使用

### 需要安装的插件
- Advanced Slides
- Image auto upload Plugin
- Projects

### 安装方法
- 找到一个已经安装了这三个插件的同学
- 要到note文件夹下的隐藏文件夹 .Obisdian 下的 plugins 文件夹
- 在自己的note文件夹下创建 .Obisdian 文件夹并将 plugins 文件夹放进去
- 重启Obsidian
- 在 plugins 文件夹左下角的设置中的第三方插件选项栏中启用这三个插件

### 插件功能

#### Advanced Slides
- 将md文件以类似PPT的形式放映
	- 使用 --- 作为每页PPT间的分隔符
	- 在需要放映的文件打开后点击左侧工具栏中的Show Slide Preview（一个显示器形状的图标）

### Image auto upload Plugin
- 自动上传放入Obsidian中的图片至图床
- 需要预先安装 PicGo 并启动
	- 下载地址: https://picgo.github.io/PicGo-Doc/
	- 设置： 图床设置 / 腾讯云COS （SecretKey 请咨询： [[史欣]] ) ![PicGo-2.3.1-raser-setup.png](https://raser-1314796952.cos.ap-beijing.myqcloud.com/media/PicGo-2.3.1-raser-setup.png)
	- 如上图设置好后，点击“确定” 和“设为默认图床”按钮： ![PicGo-Setup-Button.png](https://raser-1314796952.cos.ap-beijing.myqcloud.com/media/PicGo-Setup-Button.png)

### Projects
- 用于总体查看note中部分文件的更新时间、状态等信息

# 使用VScode登录服务器

首先确保自己已经获得AFS账号再进行以下操作

## 使用vscode作为客户端登录
- 安装remote-vscode插件
- 左边会出现remote栏
- ![](https://raser-1314796952.cos.ap-beijing.myqcloud.com/media/%E6%88%AA%E5%B1%8F2023-01-12%2010.57.01.png)
- 在SSH栏中添加SSH链接
	- `ssh 用户名@lxslc7.ihep.ac.cn`
- 会弹出一个窗口，对应访问服务器的vscode，没弹出就手动从左下角连接
- ![](https://raser-1314796952.cos.ap-beijing.myqcloud.com/media/%E6%88%AA%E5%B1%8F2023-01-12%2011.03.42.png)
- ![](https://raser-1314796952.cos.ap-beijing.myqcloud.com/media/%E6%88%AA%E5%B1%8F2023-01-12%2011.04.12.png)
- 成功连接服务器后会让你输密码，第一次登录会在服务器上下载vscode-server
- 如果vscode让你手动选服务器操作系统，选linux
- 进入服务器后可以打开文件夹或者从终端访问文件

## 防止爆盘
- 每个人服务器home下分配的空间有限，vscode的服务器端文件较大，需要使用软连接将vscode放到其他磁盘
- 使用终端登录服务器（本地的）
	- `ssh 用户名@lxslc7.ihep.ac.cn`
- 将.vscode-server移动到其他硬盘：
	- `mv .vscode-server /workfs2/用户组/用户名/`（用户组是申请AFS账号是选择的，atlas）
- 创建一个软连接
	- `ln -s /workfs2/用户组/用户名/.vscode-server .`
		- 注意后面的空格和点
- `ls -a`，如果有浅蓝色软链接就代表成功
- 使用VScode登录服务器
